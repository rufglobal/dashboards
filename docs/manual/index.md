## Getting Started

### What This Manual Covers
This guide teaches you how to work effectively with AI agents - the smart assistants that help you get work done. You'll learn the key concepts that make the difference between expensive, frustrating interactions and smooth, cost-effective collaboration.

### Prerequisites
- Access to one or more AI agents through your organization
- Basic understanding that agents are AI assistants that can help with various tasks

### Quick Start
1. **Start a conversation** - Message an agent with a specific request
2. **Be clear about what you want** - The more specific, the better results
3. **Keep related requests in the same conversation** - Saves money and context
4. **Ask for explanations** - Agents can teach you better ways to work with them

## What are Skills

### What are skills?
Skills are like specialized apps for your agents. Just as you install apps on your phone to add new features, agents use skills to learn specialized tasks they couldn't do before.

**Think of it this way:**
- Basic agent = smartphone out of the box
- Skills = apps you install to add specific capabilities  
- Each skill teaches an agent how to do something specific really well

**Examples of skills:**
- **PDF skills** - Create, edit, merge, split PDF documents
- **Calendar skills** - Schedule meetings, manage appointments
- **Data analysis skills** - Analyze spreadsheets, create charts and reports
- **Marketing skills** - Create campaigns, write copy, manage social media
- **Coding skills** - Write, debug, and deploy software

### Why you should use skills

#### Faster, better results
Without skills, you get generic responses. With skills, you get professional-quality output tailored to your specific needs.

**Example comparison:**

❌ **Without skills:** 
- You: "Help me with this data"
- Agent: "I can see you have some numbers. What would you like to do with them?"
- Result: Back-and-forth questions, generic advice

✅ **With data analysis skill:**
- You: "Help me with this data" 
- Agent: Uses skill → Creates charts, identifies trends, provides business insights
- Result: Executive-ready analysis in minutes

#### Consistent quality
Skills contain best practices and proven approaches. You get the same high-quality output every time, without having to explain processes repeatedly.

#### Specialized expertise
Each skill contains deep knowledge about specific domains. It's like having access to experts in every field.

### How to use skills

Skills work automatically - you don't need to install or activate them. Just ask for what you need, and the agent will use the right skills behind the scenes.

#### Basic Usage
```
"Create a PDF report from this spreadsheet data"
```
↳ Agent automatically uses PDF creation and data analysis skills

```
"Schedule a team meeting for next week"  
```
↳ Agent automatically uses calendar management skills

#### Advanced Usage - Being specific
You can mention specific approaches or tools you want:

```
"Use GitHub to create a new issue for this bug report"
```
↳ Explicitly uses GitHub integration skills

```
"Create a marketing email campaign using our brand guidelines"
```  
↳ Uses marketing skills with brand compliance

## What are Sessions

### What are sessions?
A session is a conversation thread with an agent. Every time you start a new conversation, you create a new session. Think of it like text message threads - each separate conversation is its own session.

**Simple analogy:**
- Session = phone call or text conversation thread
- Starting fresh conversation = new session
- Continuing same conversation = same session

**Visual example:**
```
Session 1: "Analyze this data" → Agent responds → You: "Now create a chart" 
Session 2: [2 hours later] "Send me the report" (new conversation)
```

### Why you need to know about sessions

Sessions directly control your costs. This is the most important concept for managing expenses when working with agents.

#### How sessions affect costs
- **New session** = Agent loads its full "brain" from scratch (expensive)
- **Continuing session** = Agent remembers your previous conversation (much cheaper)
- **More sessions** = Higher bills

#### The cost difference is huge
- New session: Agent loads all its knowledge, skills, and context = $2-5 per session start
- Continuation: Agent just processes your new message = $0.10-0.50 per message

**Real example:** 
- 10 separate sessions = $20-50 in startup costs alone
- 1 session with 10 messages = $1-5 total

### How to optimize session costs

#### Keep related conversations together

❌ **Expensive approach (creates multiple sessions):**
```
11:00 AM: "Analyze our sales data"
[Agent responds]
[You close conversation]

2:00 PM: "Create a chart from that analysis"  
[Agent responds - had to reload context]
[You close conversation]

4:00 PM: "Email the chart to my team"
[Agent responds - had to reload context again]
```
**Cost:** 3 sessions × $3 each = $9 in startup costs

✅ **Efficient approach (single session):**
```
11:00 AM: "Analyze our sales data, create a chart, and email it to my team"
[Agent does all three tasks in one conversation]
```
**Cost:** 1 session × $3 = $3 total

#### Batch similar requests

❌ **Expensive:**
```
"Send an email to John about the meeting"     (Session 1)
"Send an email to Sarah about the meeting"   (Session 2)  
"Send an email to Mike about the meeting"    (Session 3)
```

✅ **Efficient:**
```
"Send emails to John, Sarah, and Mike about tomorrow's meeting. Include the agenda and Zoom link."
```

#### Plan ahead and give complete context

❌ **Expensive (multiple sessions because of missing info):**
```
"Create a report"
Agent: "What kind of report? What data should I use?"
"Use the sales data from last month"  
Agent: "What format do you want?"
"PowerPoint presentation"
Agent: "Who is the audience?"
"The executive team"
```

✅ **Efficient (everything in one session):**
```
"Create a PowerPoint presentation analyzing last month's sales data for the executive team. Include trends, key insights, and recommendations."
```

### Real examples of session optimization

#### Example 1: Daily check-ins
❌ **Expensive pattern:** Asking for status updates on different projects throughout the day
```
9:00 AM: "How's Project A going?" (Session 1)
11:00 AM: "Any updates on Project B?" (Session 2)
2:00 PM: "What's the status of Project C?" (Session 3)  
4:00 PM: "How are we doing on Project D?" (Session 4)
```
**Cost:** 4 sessions = $12-20

✅ **Efficient pattern:** Single comprehensive check-in
```
9:00 AM: "Give me a status update on Projects A, B, C, and D. Include any blockers, progress updates, and next steps."
```
**Cost:** 1 session = $3-5

**Savings:** 75% cost reduction

#### Example 2: Research and presentation project
❌ **Expensive pattern:** Breaking work into separate requests
```
Day 1: "Research competitor pricing" (Session 1)
Day 2: "Find market size data" (Session 2)  
Day 3: "Create slides about pricing" (Session 3)
Day 4: "Add market data to slides" (Session 4)
Day 5: "Format presentation professionally" (Session 5)
```
**Cost:** 5 sessions = $15-25

✅ **Efficient pattern:** Comprehensive project request
```
Day 1: "Create a presentation analyzing our competitive position. Include competitor pricing research, market size data, and our positioning strategy. Format it professionally for the board meeting."
```
**Cost:** 1-2 sessions = $3-6

**Savings:** 80% cost reduction

#### Example 3: Content creation workflow
❌ **Expensive pattern:** Step-by-step requests
```
"Write a blog post about AI trends" (Session 1)
"Make it more SEO-friendly" (Session 2)
"Add some statistics" (Session 3)  
"Create social media posts to promote it" (Session 4)
"Design a newsletter featuring the post" (Session 5)
```
**Cost:** 5 sessions = $15-25

✅ **Efficient pattern:** Complete content package
```
"Create a complete content package about AI trends: SEO-optimized blog post with statistics, 5 social media promotion posts, and a newsletter featuring the article."
```
**Cost:** 1-2 sessions = $3-6

**Savings:** 80% cost reduction

### Pro tips for session management

#### Plan your full request upfront
Before messaging an agent, think about everything you'll need:
- What's the end goal?
- What information will they need?
- What format do you want?
- Who's the audience?
- When do you need it?

#### Use follow-up questions strategically  
It's okay to continue conversations with clarifications:
```
"That's great! Can you also add a summary slide at the beginning?"
"Perfect. Now can you create a shorter version for social media?"
```

#### Set clear context at the start
```
✅ Good opening: "I'm preparing for a board meeting next Tuesday. I need to present our Q4 performance compared to competitors. Please create a professional presentation that covers our revenue growth, market share changes, and competitive positioning. The audience is 8 executives who want data-driven insights."
```

#### Batch similar work
- Group all emails together
- Request multiple document formats at once  
- Ask for several options or alternatives in one request

#### Use natural conversation flow
Don't artificially force everything into one message if it doesn't make sense. But do keep related work in the same conversation thread.

## Best Practices

### Communicating with Agents

#### Be specific and provide context
- ✅ "Create a customer onboarding email sequence for our SaaS product. Target audience is small business owners who just signed up for our free trial."
- ❌ "Write some emails"

#### Give complete requirements upfront
- Timeline: "I need this by Friday"
- Audience: "For technical developers" or "For non-technical executives"  
- Format: "As a PDF report" or "In bullet points"
- Constraints: "Keep it under 2 pages" or "Use our brand colors"

#### Ask for what you actually want
- ✅ "Create a presentation I can deliver to the sales team"
- ❌ "Give me some information about our product"

### Working Efficiently

#### Batch related work
Instead of 5 separate requests, combine them:
- "Create slides 1-5 for my presentation"
- "Write email templates for onboarding, renewal, and cancellation"
- "Analyze data from January, February, and March"

#### Provide examples when helpful
- "Write it in the same style as this email [paste example]"
- "Use the same format as our previous reports"
- "Make it sound like our CEO's blog posts"

#### Ask for alternatives  
- "Show me 3 different approaches"
- "Give me options ranging from conservative to aggressive"
- "What are the pros and cons of each method?"

### Getting Better Results

#### Request explanations
- "Why did you choose this approach?"
- "What best practices are you following?"
- "How does this compare to industry standards?"

#### Iterate in the same session
- "That's good, but can you make it more formal?"
- "Perfect! Now create a shorter version for executives"
- "Great approach. Let's apply this same method to our other products"

#### Build on previous work
- "Use the same analysis method for Q2 data"
- "Apply these insights to our marketing strategy"  
- "Create similar reports for our other product lines"

## Cost Optimization

### Understanding Your Costs

#### Session startup costs are the biggest expense
- New session = $2-5 regardless of how much work gets done
- Message in existing session = $0.10-0.50
- **Key insight:** Minimize new sessions, maximize work per session

#### Model costs vary significantly  
- **Sonnet (standard):** Good for most tasks, cost-effective
- **Opus (premium):** 5x more expensive, use only for complex reasoning
- **Tip:** Ask for Sonnet unless you need advanced analysis

### Optimization Strategies

#### 1. Plan comprehensive requests (80% savings potential)
Instead of 10 separate conversations, plan one comprehensive request covering all your needs.

#### 2. Use follow-up questions (Free after session start)
Once a session is started, additional questions in the same conversation are very cheap.

#### 3. Batch similar work (75% savings)
- Group all emails together
- Process multiple documents at once
- Analyze several time periods in one request

#### 4. Provide complete context upfront (Prevents expensive back-and-forth)
Include audience, format, timeline, and constraints in your first message.

#### 5. Ask for multiple deliverables (Maximize session value)
"Create a report, executive summary, and presentation slides" costs about the same as just "create a report."

### Monitoring Your Usage

#### Check the usage dashboard regularly
- Track daily spending patterns
- Identify which agents cost the most
- Spot inefficient usage patterns

#### Look for these warning signs
- Many sessions with the same agent in one day
- High costs without proportional work output
- Repetitive requests that could be batched

#### Monthly optimization review
- Which tasks generated the most sessions?
- What requests could have been combined?
- Are you using the right models for each type of work?

## FAQ

### General Usage

**Q: How do I know which skills an agent has?**
A: Just ask! Say "What can you help me with?" or describe your task and the agent will use appropriate skills automatically.

**Q: Can I request specific skills?**
A: Yes. You can say "Use the GitHub skill to create an issue" or "Apply SEO writing best practices."

**Q: What if an agent doesn't have the skill I need?**
A: Agents can often accomplish tasks in different ways. Describe what you want to achieve and let them suggest alternatives.

### Sessions and Costs

**Q: How long does a session last?**
A: Sessions stay active as long as you keep the conversation going. There's no automatic timeout - you control when it ends.

**Q: Should I end sessions manually?**
A: No need. Sessions naturally end when you stop responding. Just start new conversations in the same thread when you have follow-up work.

**Q: Is it expensive to ask clarifying questions?**
A: No! Once a session is started, follow-up questions are very cheap. Ask as many clarifying questions as you need.

**Q: How can I tell if I'm starting too many sessions?**
A: Check your usage dashboard. If you see many sessions with the same agent in one day, you might benefit from batching requests.

### Troubleshooting

**Q: The agent seems to have "forgotten" our previous conversation**
A: You might have started a new session accidentally. Try to keep related work in the same conversation thread.

**Q: I'm getting generic responses instead of specialized help**
A: Be more specific about what you need. Instead of "help with marketing," try "create a 6-week email campaign for product launch."

**Q: My costs seem high for the work I'm getting**
A: You're likely creating too many sessions. Try batching requests and providing complete context upfront.

**Q: How do I know if I should use Opus vs Sonnet?**
A: Use Sonnet (default) for most tasks. Only request Opus for complex reasoning, analysis of ambiguous situations, or creative problem-solving.

### Best Practices

**Q: Should I be polite to agents?**
A: Be clear and direct. Agents don't need social pleasantries, but being specific about what you want helps them help you better.

**Q: Can I interrupt an agent while it's working?**
A: Yes, you can add clarifications or changes mid-task. Just continue the conversation normally.

**Q: What's the best way to give feedback on agent responses?**
A: Be specific: "That's great, but can you make it more formal?" or "Perfect style - use this approach for future requests."
